    • fr.gpmsi.poi.PoiHelper 
    • public class PoiHelper
extends java.lang.Object

      Classe avec méthodes utilitaires pour l'utilisation de fonctions Apache POI pour Excel. Les classes statiques sont utilisables directement, les autres nécessitent de créer un objet qui crée un DataFormatter (ce qui permet un cache des valeurs de formats) et/ou le champ newJavaTimeUsed.

        • public static void copyStyles(org.apache.poi.ss.usermodel.Cell srcCell, org.apache.poi.ss.usermodel.Cell destCell, org.apache.poi.ss.usermodel.CellCopyContext context)

          Copier le style de srcCell vers destCell. Le org.apache.poi.ss.usermodel.CellCopyContext fourni sert à garder trace des styles qui ont déjà été clonés, pour pouvoir les réutiliser et ainsi éviter qu'ils ne soient recréés inutilement. Cette méthode est faite à partir de contenu extrait de CellUtil#copyCell(Cell, Cell, org.apache.poi.ss.usermodel.CellCopyPolicy, CellCopyContext) qui pour l'instant est encore en beta. Si la source ou la destination sont nulles, il n'y a pas d'exception lancée, mais un avertissement est envoyé sur le log.

          Parameters:
          srcCell - cellule source
          destCell - cellule destination
          context - contexte de copie

        • public final java.lang.Object getCellValueAsObject(org.apache.poi.ss.usermodel.Cell c)

          Prendre la valeur de la cellule en tant qu'objet. Ici cette méthode n'est pas statique car elle dépend de newJavaTimeUsed. Fait la différence entre les nombres et les dates. Les objets qui peuvent être retournés :

          • CellType._NONE
          • CellType.BLANK
          • Boolean
          • Date (java.util) (si newJavaTimeUsed est false
          • LocalDateTime (si newJavaTimeUsed est true
          • Double
          • String (en cas de type non géré, renvoie c.toString() mais c'est un bug si cela arrive)
          • Byte (en cas de cellule de type erreur)
          • null
          A noter que si la cellule est de type FORMULA (une formule Excel) c'est la dernière valeur évaluée au moment de la sauvegarde qui est retournée. Les formules ne sont pas réévaluées ici.
          Parameters:
          c - La cellule à lire, peut être null.
          Returns:
          La valeur objet de la cellule. Peut retourner null.

        • public static final java.lang.String getCellValueAsString(org.apache.poi.ss.usermodel.Cell c, java.text.DateFormat df, java.text.NumberFormat nf)

          Prendre la valeur de la cellule, garantit qu'aucune valeur null ne sera retournée. Si la cellule est de type Date, utilise le dateFormat fourni. Si la cellule est de type Number, utiliser le NumberFormat fourni.

          Parameters:
          c - La cellule à lire, peut être null.
          df - Le format de date à utiliser, s'il est null, le format par défaut de java sera utilisé.
          nf - Le format de nombre à utiliser, s'il est null, le format par défaut de java sera utilisé.
          Returns:
          La valeur textuelle de la cellule. Au lieu de null si c'est le cas, retourne la chaîne vide "".

        • public java.lang.String getCellValueAsString(org.apache.poi.ss.usermodel.Cell c)

          Prendre la valeur de la cellule, garantit qu'aucune valeur null ne sera retournée. A noter que ici les valeurs apparaîtront telles qu'elles sont visible dans Excel. Pour les cellules qui contiennent des formules, c'est la dernière valeur qui a été stockée par Excel qui sera retournée. Attention les cellules vierges de type BLANK renvoient le texte "BLANK" !

          Parameters:
          c - La cellule à lire, peut être null.
          Returns:
          La valeur textuelle de la cellule. N'est jamais null.

        • public static int getColumnNumber(java.lang.String columnLetters)

          Calculer le numéro de colonne à partir de la colonne en lettres. Exemple : 

           getColumnNumber("A"); //retourne 1
 getColumnNumber("Z"); //retourne 26
 getColumnNumber("AA"); //retourne 27
 getColumnNumber("AAA"); //retourne 703 (1 + 26 + 26*26
          (Si on a un doute sur un numéro, taper dans Excel la formule =COLONNE() on aura le numéro de colonne Excel)
          Parameters:
          columnLetters - La colonne en lettres.
          Returns:
          Le numéro de colonne pour Poi (part de 0, c'est celui de Excel - 1)
