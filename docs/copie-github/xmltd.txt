                         XMLTD XML Table Data
                         
                         
L'import et l'export de données est peu fiable en csv, notamment à cause de
toutes les variations possibles dans les formats de date, de nombre, etc.

Cette partie est donc un ensemble de conventions pour exporter une table
dans un fichier XML avec une structure rigide.
Chaque rangée est dans un élément "r".
Chaque colonne est stockée dans un attribut de "r". La première colonne est
dans l'attribut "a", la deuxième colonne dans l'attribut "b", etc. 
Quand on est arrivé à z, on recommence avec aa, ab, etc. (comme dans Excel).
De cette manière, on n'a que 4 caractères supplémentaires par colonne (contre
1 pour le format csv)

Exemple :
<xmltd>
<column-defs>
  <column letters="a" name="MyNumber" type="integer"/>
  <column letters="b" name="Greeting" type="varchar" max-size="64"/>
  <column letters="c" name="GreetingDate" type="date"/>
</column-defs>
<table name="MyGreetings">
<r a="3.14"b="Bonjour !"c="deux&#10;lignes"d="2019-20-02"/>
</table>
</xmltd>

Les types de données :
varchar : pour tous les types de textes
integer : entier signé : [+-]?[0-9]+
real : nombre réel [+-][0-9]+[.]?[0-9]*[Ee]?[+-]?[0-9]*
date : date au format iso, ex : 2019-12-24
time : heure au format iso, ex : 12:30:45.23
datetime : dat et heure au format iso, séparés par un T, ex : 2019-12-24T12:30:45.23

Les attributs :
max-size : nombre
allow-empty : 1 ou 0

L'idée est que l'on puisse recréer une table dans le même système qui a 
écrit ces données (par défaut la base H2 pour nous).
On peut donc inclure des informations supplémentaires pour cela, notamment
l'attribut external-declaration
Un élément supplémentaire est external-declarations pour ajouter du script
supplémentaire comme de la création d'index. On peut en rajouter plusieurs ce
qui permet de reconstituer une déclaration de table complète avec les contraintes
et les créations d'index.
