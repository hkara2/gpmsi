Conventions CIM 10 CoCoa

A15.0 equivalent a A150

C73 sans rien derriere signifie "juste C73"

S72.- signifie tous les codes qui commencent par S72.

Ambibuité : on peut aussi dire "se codent avec la catégorie S72"

N00-N08 : tous les codes qui vont de N00 à N08

Utilisation d'un caractère à la 5ème position alors que le code d'avant ne fait 
que 3 caractères : on utilise + comme caractère de remplissage. Ex :

M45.+7
J96.1+0
J96.1+1

M62.8-0 : le trait d’union
remplace les caractères 0 à 9 : M62.800, M62.810, M62.820, M62.830, M62.840, 
M62.850, M62.860, M62.870, M62.880, M62.890)

Utilisation dans pmsixml

Taille
La taille officielle des diagnostics utilisés dans les RSS est de 8 caractères.
Donc un code CIM-10 sera agrandi à 8 caractères en ajoutant des espaces à la fin.

Traitement

Tous les caractères '.' sont supprimés

Le caractère '_' est générique pour 1 caractère.

Le caractère '%' est générique pour plusieurs caractères.

Par défaut, le caractère '%' est implicitement ajouté à la fin. Par exemple,
S72 signifie en réalité S72%

Si on ne veut pas cela, il faut faire finir le code par !
Exemple : S72!

Exemple :

Tous les codes qui commencent par J96.1 :
J96.1

Tous les codes qui commencent par J96.1+ :
J96.1+
(cela exclut donc le code J96.1 seul)

Le code J96.1 seul :
J96.1!

Intervalles de codes

code de départ ':' code de fin

Inclut tous les codes qui satisfont à la condition suivante :

code >= code de départ

code < code de départ complété avec des caractères '9'

N.B. le code '+' est plus petit que le code '0', donc N03.+1:N04 incluera
les codes NO3.+1 N03.+2 etc. mais aussi N03.01 N03.1 etc. 

En pratique les intervalles sont surtout composés de préfixes à 3 chiffres,
comme C00:C97 (les tumeurs malignes),
D00:D09 (tumeurs in situ), D10:D36 (Tumeurs bénignes),
D37:D48 (Tumeurs à évolution imprévisible ou inconnue),
etc.
